## File Summaries

### Core System (Backend)

### `cns.py`
**Role:** The main orchestrator (Central Nervous System) that manages the concurrent agent threads (Hunter, Analysts) and bootstraps the system.
**Key Exports:**
- `hunter_loop()`: Void - Continuously polls the agenda, generates strategies, and dispatches job scraping sweeps.
- `analyst_loop(worker_id)`: Void - Consumes scraped jobs from the queue, evaluates them via the LLM, and triggers resume generation.
- `status_loop()`: Void - Periodically exports system queue telemetry.
- `job_queue` - Thread-safe queue holding newly scraped jobs pending AI analysis.
**Dependencies:** `config`, `tools.memory`, `tools.senses`, `tools.brain`, `tools.server`

### `config.py`
**Role:** The global configuration manifest dictating system parameters, LLM model selection, and domain-specific search heuristics.
**Key Exports:**
- `OLLAMA_MODEL` / `HEAVY_MODEL` - Defines the local LLMs used for scouting and asset generation.
- `INITIAL_SEARCH_TERMS` - The baseline array of technical search query strings.
- `TITLE_BLACKLIST` / `COMPANY_BLACKLIST` / `ALLOWLIST_OVERRIDES` - Regex arrays defining strict rejection criteria.
- `MAX_COMMUTE_MILES` / `MIN_SCORE` - Operational threshold constraints.
**Dependencies:** None

### `tools/brain.py`
**Role:** The LLM interface layer handling job evaluation, data extraction, and dynamic zero-hallucination resume synthesis.
**Key Exports:**
- `Brain(class)` - Encapsulates Ollama communication and prompt logic.
- `Brain.parse_command(user_input): string[]` - Translates natural language commands into actionable search arrays.
- `Brain.evaluate(job): dict` - Extracts structured JSON metadata and assigns a technical score to a raw job description.
- `Brain.build_jit_resume(job): tuple` - Generates a targeted Markdown resume tailored to the job description constraints.
- `Brain.strategize(job): string[]` - Identifies adjacent search vectors based on highly-scored targets.
**Dependencies:** `ollama`, `config`, `tools.memory`

### `tools/memory.py`
**Role:** The state persistence and database manager interacting with SQLite for job histories, geocoding caching, and telemetry.
**Key Exports:**
- `Memory(class)` - Provides an abstraction over the `targets.db` SQLite database.
- `Memory.get_or_fetch_distance(loc_str): float` - Calculates Haversine distance using a cached Nominatim API call.
- `Memory.save_job(job): boolean` - Persists scraped jobs, rejecting duplicates.
- `Memory.update_job(...)` - Updates job records with LLM insights and modifies queue status.
- `Memory.add_to_agenda(terms)` - Queues search strategies for the Hunter thread.
- `Memory.get_global_stats(): dict` - Aggregates pipeline metrics for the UI dashboard.
**Dependencies:** `sqlite3`, `config`

### `tools/senses.py`
**Role:** The external data ingestion layer responsible for executing web scraping operations against job boards.
**Key Exports:**
- `perform_sweep(term, location, is_remote, batch_size, offset): dict[]` - Executes the JobSpy scraper and returns normalized job payloads.
**Dependencies:** `jobspy`, `pandas`

### `tools/server.py`
**Role:** The embedded HTTP server providing a REST API for the React UI and housing the Markdown-to-PDF compilation engine.
**Key Exports:**
- `start_server(): void` - Bootstraps the local threaded HTTP daemon on port 8050.
- `convert_to_pdf(md_path, output_path, target_role_arg): boolean` - Transforms generated Markdown resumes into styled PDFs using `xhtml2pdf`.
- `CRMHandler(class)` - Dispatches UI requests (e.g., `/api/jobs`, `/api/deploy`, `/api/rescore`) to the `Memory` and `Brain` modules.
**Dependencies:** `http.server`, `markdown2`, `xhtml2pdf`, `tools.memory`, `tools.brain`

### Maintenance & Revision Tools

### `apply_heuristics.py`
**Role:** Scans the active job database to bulk-reject roles that violate updated domain blacklists.
**Key Exports:**
- `check_heuristics(title, company): string | None` - Returns a violation reason if blacklisted constraints are met.
- `run_purge(): void` - Executes the bulk database cleanup transaction.
**Dependencies:** `tools.memory`, `config`

### `backfill_distances.py`
**Role:** Iterates over legacy database records to populate missing Haversine distance values via the geocoding engine.
**Key Exports:**
- `backfill(): void` - Fetches and saves coordinates for unmapped locations.
**Dependencies:** `tools.memory`

### `backfill_insights.py`
**Role:** Re-evaluates unprocessed jobs through the heavy LLM to backfill structured JSON constraints (tech stack, salary, clearance).
**Key Exports:**
- `run_backfill(mode): void` - Iterates pending jobs, pings the Brain module, and updates SQLite.
**Dependencies:** `tools.memory`, `tools.brain`

### `clean_data.py`
**Role:** A destructive utility to wipe the SQLite database, queue files, and reset the agent to factory zero.
**Key Exports:**
- `purge_data(): void` - Deletes all core data files.
- `flush_agenda(): void` - Clears pending items from the execution queue.
**Dependencies:** `config`

### `revision-tools/export_logs.py`
**Role:** Dumps the mission and search yield logs to a CSV file for external analytics.
**Key Exports:**
- `export(): void` - Writes `data/mission_history.csv`.
**Dependencies:** `tools.memory`, `csv`

### `revision-tools/migrate_db_filenames.py`
**Role:** Corrects database pointers after legacy resume markdown files are batch-renamed.
**Key Exports:**
- `migrate(): void` - Updates the `selected_resume` column in SQLite.
**Dependencies:** `config`, `sqlite3`

### `revision-tools/rebuild_missing_experience.py`
**Role:** Scans generated resumes and forces the injection of dropped historical job roles to prevent timeline gaps.
**Key Exports:**
- `run_backfill(): void` - Modifies existing `.md` files and recompiles PDFs.
**Dependencies:** `config`, `tools.server`

### `revision-tools/rebuild_missing_stealth.py`
**Role:** Forcibly injects a specific "Stealth Platform" employment block into resumes missing current-role coverage.
**Key Exports:**
- `run_backfill(): void` - Modifies `.md` contents and triggers PDF recompilation.
**Dependencies:** `config`, `tools.server`

### `revision-tools/rebuild_pdfs.py`
**Role:** Batch converts all existing Markdown files into styled PDFs, primarily used after CSS template updates.
**Key Exports:**
- `run_rebuild(): void` - Triggers `convert_to_pdf` across the `output_mds` directory.
**Dependencies:** `tools.server`

### `revision-tools/rebuild_skill_titles.py`
**Role:** Performs fast string replacements to correct known capitalization errors in markdown headers.
**Key Exports:**
- `run_backfill(): void` - Rewrites `.md` files matching bad strings.
**Dependencies:** `config`, `tools.server`

### `revision-tools/rebuild_summaries.py`
**Role:** Uses the LLM to rewrite only the professional summary section of existing resumes without altering the strict facts below it.
**Key Exports:**
- `regenerate_summary(target_title, company, current_summary): string` - Prompts the LLM for a revised summary.
- `run_backfill(): void` - Parses `.md` files and applies summary replacements.
**Dependencies:** `config`, `tools.server`, `ollama`

### `revision-tools/reweigh_queue.py`
**Role:** Forces a system-wide re-evaluation of target jobs against the latest scoring rules, dropping items that fall below the threshold.
**Key Exports:**
- `run_assimilation(): void` - Iterates active jobs through `Brain.evaluate`.
**Dependencies:** `tools.memory`, `tools.brain`, `config`

### `test_strategy.py`
**Role:** Sandbox script for verifying the LLM's strategy initialization logic.
**Key Exports:** None.
**Dependencies:** `tools.brain`

### `verify_db.py`
**Role:** CLI inspector tool that verifies SQLite schema integrity and outputs mission yield metrics to the terminal.
**Key Exports:**
- `verify_missions(): void` - Runs `PRAGMA` checks and prints the table payload.
**Dependencies:** `sqlite3`

### UI Frontend (React)

### `web/src/app.jsx`
**Role:** The root application wrapper providing the global layout, navigation sidebar, and Agent power controls.
**Key Exports:**
- `Layout`: Component - Wraps content, manages the Sidebar, and polls the Python API for agent status.
- `App`: Component - Registers routing for `/`, `/board`, and `/profile`.
- `isActive` - State tracking whether the backend scraping loop is running.
**Dependencies:** `react-router-dom`, `lucide-react`, `./pages/*`

### `web/src/pages/feed.jsx`
**Role:** Renders the live incoming job feed and commander terminal for dispatching new Hunter orders.
**Key Exports:**
- `Feed`: Component - Mounts the search bar, filter controls (Remote, Score, Stack), and maps over the job list.
- `jobs` - State holding the live array of target opportunities.
- `command` - State capturing natural language inputs to send to the Agent's agenda.
**Dependencies:** `../components/JobCard`, `../components/JobModal`, `lucide-react`

### `web/src/pages/board.jsx`
**Role:** A Kanban board view organizing job applications by their current lifecycle state (Target, Applied, Interview, Offer).
**Key Exports:**
- `Board`: Component - Renders swimlanes mapping the `filteredJobs` array.
- `jobs` - State containing the synced SQLite job rows.
**Dependencies:** `../components/JobCard`, `../components/JobModal`, `lucide-react`

### `web/src/pages/profile.jsx`
**Role:** A telemetry dashboard visualizing system KPIs, pipeline distributions, and agent log history.
**Key Exports:**
- `Profile`: Component - Fetches `/api/stats` and `/api/logs` and renders them via Recharts.
- `stats` / `logs` - State holding aggregated database metrics.
**Dependencies:** `recharts`

### `web/src/components/jobcard.jsx`
**Role:** Displays a summarized, actionable card for a single job opportunity.
**Key Exports:**
- `JobCard`: Component - Visualizes score, location, dynamic tech-stack pills, and triage buttons.
**Dependencies:** `lucide-react`, `react`

### `web/src/components/jobmodal.jsx`
**Role:** A rich, 3-pane resizable interface displaying the raw job description, the AI's tactical analysis, and the generated Markdown resume.
**Key Exports:**
- `JobModal`: Component - Supports keyboard shortcuts, in-place resume editing, and PDF deployment signaling.
- `resumeContent` - State holding the fetched `.md` text for the current job.
- `editContent` - State managing the text buffer when the user actively overrides the LLM's generated asset.
**Dependencies:** `lucide-react`, `react`

### `web/vite.config.js`
**Role:** Build tool configuration defining local development proxies.
**Key Exports:**
- Default Export - Vite configuration mapping `/api` and `/data` to `http://127.0.0.1:8050` (the Python backend).
**Dependencies:** `@vitejs/plugin-react`
